# 커밋 및 진행상황 저장

## 개요

작업 완료 후 자동으로 git 커밋하고 Progress.md에 현재 상황을 저장합니다.

## 실행 조건

사용자가 다음 중 하나를 요청할 때:
- "커밋해줘"
- "저장해줘"
- "현재 상황 저장해줘"
- 작업 완료 후

## 실행 단계

### 1. Git 상태 확인
```bash
git status
git diff --stat
git log -3 --oneline
```

### 2. 변경된 파일 분석
- 변경된 파일 목록 수집
- 변경 유형 분석 (새 파일, 수정, 삭제)
- 관련된 작업 파악

### 3. 커밋 메시지 작성
변경 내용에 따라 자동으로 커밋 메시지 작성:
- **feat**: 새로운 기능 추가
- **fix**: 버그 수정
- **refactor**: 코드 리팩토링
- **docs**: 문서 업데이트
- **chore**: 설정 변경 등

예시: `feat: ScriptableObject Addressables 자동 설정 스크립트 추가`

### 4. Git 커밋
```bash
git add -A
git commit -m "<커밋 메시지>"
```

### 5. Progress.md 업데이트

진행 중인 작업이나 완료된 작업이 변경되었다면 Progress.md를 업데이트합니다.

#### 업데이트 섹션 선택:
- **완료된 작업**: 작업이 완료되면 Completed 섹션에 추가
- **진행 중인 작업**: 현재 작업 중인 내용 업데이트
- **발견 사항**: 새로운 발견이 있으면 Discoveries에 추가
- **다음 단계**: 계획이 변경되면 업데이트

#### 업데이트 형식:
```markdown
### [작업 제목] ✅
- 세부 항목 1
- 세부 항목 2
```

### 6. 결과 보고
커밋 해시와 변경된 파일 목록을 사용자에게 보고

## 예시 실행

**사용자**: "작업 완료, 커밋해줘"

**실행**:
1. `git status` → 변경 파일 확인
2. 커밋 메시지 작성: `feat: DI 트러블슈팅 스킬 추가`
3. `git add -A && git commit -m "..."`
4. Progress.md 업데이트 (필요시)
5. 결과 보고: 커밋 해시 + 변경 파일 목록

## 주의사항

- 커밋 전 사용자에게 확인 요청하지 않음 (사용자가 직접 요청한 경우)
- 커밋 메시지는 영어로 작성 (일관성 유지)
- Progress.md는 한국어로 작성 (기존 형식 유지)
- 민감한 파일(.env, credentials.json 등)은 커밋에서 제외

## 관련 파일

- `Progress.md` - 프로젝트 진행상황
- `Setup.md` - Unity 수동 설정 가이드
- `.opencode/skills/di-troubleshooting.md` - DI 트러블슈팅 가이드
