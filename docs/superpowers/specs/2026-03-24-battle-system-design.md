# 전투 시스템 설계

## 개요

턴제 전투 시스템. 플레이어와 몬스터가 번갈아가며 행동. 전투 로그로 모든 행동 표시.

## 핵심 기능

### 1. 턴 시스템
- 플레이어 턴 → 몬스터 턴 번갈아 진행
- 속도(Speed) 스탯으로 선공 결정
- 턴 시작 시 상태이상 처리 (데미지, 지속시간 감소)

### 2. 행동별 턴 소모
| 행동 | 턴 소모 |
|------|---------|
| 공격 | O |
| 스킬 | O |
| 방어 | O |
| 소모품 사용 | O |
| 장비 장착/해제 | X |
| 인벤토리 열기 | X |

### 3. 몬스터 AI
- 기본 공격만 사용 (MonsterData.Attack)
- 상태이상 있는 몬스터: 공격 시 확률적 상태이상 적용
- 보스: 받는 데미지 -2

### 4. 전투 로그 형식
```
[플레이어의 턴]
플레이어 공격 → 25 데미지
(상태이상) 몬스터 화상 적중

[몬스터의 턴]
몬스터 일반 공격
플레이어 → 10 데미지

[플레이어의 턴]
플레이어 스킬 발동: 십자베기
몬스터 → 35 데미지

[플레이어의 턴]
플레이어 아이템 사용: 회복 포션
플레이어 HP +50 회복

[플레이어의 턴]
플레이어 버프 발동: 공격력버프 (3턴간 30% 증가)
```

### 5. 사망 이벤트
- 몬스터 사망: 승리 로그, 골드 획득, 처치 수 증가
- 플레이어 사망: 패배 로그, 게임오버

## 구현 범위

### 수정 파일
- `BattleManager.cs`: 몬스터 턴 처리, 전투 로그 이벤트
- `BattleController.cs`: 로그 UI 연결, 사망 처리
- `BattleLogUI.cs`: 로그 포맷팅

### 새 파일
- 없음 (기존 구조 활용)

## 작업 목록

1. 몬스터 턴 구현 (기본 공격)
2. 전투 로그 시스템 연결
3. 사망 이벤트 처리
4. 턴 소모 로직 (아이템 사용 시)
